Cache design and eviction policy
    I decided to use an in-memory LRU cache implemented with two data structures:
        * A hash map from set_id to node, for constant-time lookup.
        * A doubly linked list, where head is most recently used and tail is least recently used.

    Eviction policy
        * When capacity is reached, the least recently used entry (tail) is removed.

Complexity
    * Lookup in cache: O(1)
    * Move existing node to head: O(1)
    * Insert new node: O(1)
    * Evict tail node: O(1)

I added the cache header to /sets endpoint like this:
    response.cache_control.max_age = 60 # seconds
    response.cache_control.public = True
